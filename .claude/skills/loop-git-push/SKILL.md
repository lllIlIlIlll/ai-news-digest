# Loop Git Push

自动循环执行 `git push`，如果没有更新则停止。

## 使用方式

用户说"循环推送"或"auto push"时激活此 skill。

## 执行步骤

1. 进入项目目录
2. 检查 `git status` 是否有未提交的更改或未推送的提交
3. 如果没有可推送的内容，输出提示并停止
4. 如果有待推送内容，后台循环执行：
   - 每 30 分钟执行一次 `git push`
   - 每次推送后检查是否有新的未推送提交
   - 如果连续两次推送都没有更新任何内容，自动停止任务
5. 报告任务状态

## 算法逻辑

```
已推送commit hash vs origin/main commit hash
如果相同 → 检查是否有未提交的更改
  如果有 → 提交并推送，然后继续循环
  如果没有 → 继续等待30分钟

如果不同（有新提交）→ 推送，然后继续循环

推送后检查：
  推送前后 commit hash 没变 → count++
  如果 count >= 2 → 停止（没有新内容）
  否则继续等待30分钟
```

## 注意事项

- 使用后台任务执行，不阻塞主线程
- 每次检查前先 `git fetch` 获取最新状态
- 推送失败时不计入"无更新"计数，而是重试
